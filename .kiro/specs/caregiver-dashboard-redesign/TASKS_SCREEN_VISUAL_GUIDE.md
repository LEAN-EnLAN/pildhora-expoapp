## Color Palette

### Task Card
- **Background:** `#FFFFFF` (colors.surface)
- **Border:** None (elevated shadow instead)
- **Shadow:** Elevation 4, opacity 0.1

### Checkbox States
- **Uncompleted Icon:** `square-outline`, `#9CA3AF` (colors.gray[400])
- **Completed Icon:** `checkbox`, `#34C759` (colors.success)
- **Touch Target:** 44x44 points

### Text States
- **Active Title:** `#111827` (colors.gray[900]), weight 500
- **Completed Title:** `#6B7280` (colors.gray[500]), weight 400, strikethrough
- **Font Size:** 16px (typography.fontSize.base)

### Buttons
- **Primary (Add Task):** `#007AFF` background, white text
- **Delete:** `#FF3B30` icon color
- **Outline (Cancel):** `#007AFF` border and text, transparent background

### Empty State

## Interaction Flow

### Adding a Task
```
1. User taps "Nueva Tarea" button
   โ
2. Modal slides up from bottom
   โ
3. User enters task description
   โ
4. User taps "Agregar" button
   โ
5. Validation runs
   โ
6. If valid: Task created, modal closes, list refreshes
   If invalid: Error message shown
```

### Completing a Task
```
1. User taps checkbox
   โ
2. Icon changes immediately
   โ
3. Text style updates (strikethrough, gray)
   โ
4. Firestore update in background
   โ
5. List refreshes with new data
```

### Deleting a Task
```
1. User taps delete button (trash icon)
   โ
2. Confirmation alert appears
   โ
3. User confirms deletion
   โ
4. Task removed from Firestore
   โ
5. List refreshes, task disappears
```

## Design Principles

1. **Simplicity:** Clean, minimal interface focused on task management
2. **Clarity:** Clear visual hierarchy and state indicators
3. **Consistency:** Matches design system used throughout app
4. **Accessibility:** Full screen reader support and proper touch targets
5. **Feedback:** Immediate visual feedback for all interactions
6. **Efficiency:** Quick task creation and completion with minimal taps

## Comparison with Other Screens

### Similarities
- Same Card component style
- Same Button variants
- Same Input styling
- Same Modal behavior
- Same color palette
- Same spacing system

### Differences
- Simpler data model (no complex relationships)
- No real-time sync requirements
- No patient association (caregiver-only)
- Minimal validation rules
- No filtering or sorting (yet)
